---
📝 Explanation of Each Folder
• app/ → Core application code.
	◦ main.py → FastAPI entry point, defines endpoints.
	◦ database/ → Handles PostgreSQL connection and query logging.
	◦ rag/ → PDF ingestion, chunking, embeddings, vector store setup.
	◦ agent/ → Agent orchestration, defines tools and LLM logic.
	◦ models/ → Pydantic models (optional, but good practice for request validation).
• data/ → Store your PDFs or other documents you want to ingest.
• chroma_db/ → Persistent storage for vector embeddings (created automatically when you run RAG).
• requirements.txt → List of dependencies to install.
• README.md → Documentation for setup and usage.
---
🧩 How It All Connects
1. User sends query → FastAPI (main.py) receives it.
2. Agent (agent.py) decides whether to retrieve from PDF or answer directly.
3. RAG (rag.py) provides relevant chunks from the PDF.
4. LLM generates the final answer.
5. Database (database.py) logs the query and answer in PostgreSQL.

🛠️ Installation Commands

1. Create and activate a virtual environment (recommended)
python -m venv venv
source venv/bin/activate   # On macOS/Linux
venv\Scripts\activate      # On Windows

2. Install FastAPI and Uvicorn
pip install fastapi uvicorn

3. Install PostgreSQL driver
pip install psycopg2

4. Install LangChain + OpenAI + ChromaDB
pip install langchain langchain-openai chromadb

5. Install PDF parsing library
pip install PyMuPDF

pip install langchain langchain-community langchain-openai
